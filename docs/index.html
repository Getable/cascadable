Cascadable UI 1.0.0

This framework is a collection of HTML and CSS components that leverages modern browser capabilities to provide developers with baselines for semantical typography, forms, buttons, containers, and grids. Cascadable is a Github Project.

Paragraphs

Paragraph text (also referred to as "body copy") looks like this. Always wrap body copy and inline phrase elements (below) in <p> tags when using CUI styles. Don't use <br> tags to separate content between elements; instead use <p> tags to separate bodies of copy, and use CSS to position other elements on the page. <br> tags should only be used to create line breaks between inline elements.

Do not use multiple <br> tags to create larger line breaks; instead, use CSS to increase the margins on the elements you wish to separate. To adjust the size of a line break within bodies of copy, use the line-height CSS property.

Phrase Elements

CUI includes styles for the following formatted phrase tags: <em> <strong>, as well as the following semantical phrase tags: <address> <abbr> <code>. To use CUI styles for inline phrase elements, simply add the cui class to the parent container.

Grids & Layout

Columns

In CUI, the grid layout is completely non-semantical. By avoiding the use of descriptive class names like .col-left, the markup will never become compromised if a developer chooses to move a .col-left element to the right side of the page — for example.

Unlike other grid systems, there is no concept of pre-defined column dimensions in CUI — which also compromise the semantic integrity of the markup. For example, if a developer chooses to make a column with the classes .four.column wider than a column with the classes .twelve.column in CSS, the column names can quickly become quite confusing.
